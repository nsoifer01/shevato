## Change 4: Wall-clock-based rest timer
### What changed
`TimerService.startRestTimer` no longer drives its countdown by decrementing a counter inside `setInterval(1000)`. Mobile browsers throttle background tabs and locked-screen tabs, which used to cause real drift — committing a set, locking the phone for 30s, and unlocking would show ~55s remaining instead of ~25s.

The timer now stores `endTime = Date.now() + duration*1000` and recomputes `remaining = ceil((endTime - Date.now())/1000)` on every tick. Tick interval increased to 4 Hz (250 ms) so the visible second-rollover stays smooth. Added `TimerService.extendRestTimer(id, extra)` so `+30s` doesn't have to stop+restart the timer; `workout-view.extendRest` now uses it.

Files touched:
- `apps/gym-tracker/js/services/TimerService.js`

## Change 5: Normalize muscle-group casing & strip dead `difficulty` field
### What changed
The exercise database used to mix two `muscleGroup` styles — lowercase-hyphenated (e.g. `'upper-pectorals'`) and Title Case (e.g. `'Upper Chest'`) — and `formatMuscleGroup` papered over both at render time. All Title Case values are now rewritten to their canonical lowercase-hyphenated equivalents. Mappings used:
- `'Chest'` → `'pectorals'`, `'Upper/Lower/Inner/Outer Chest'` → `'upper/lower/inner/outer-pectorals'`
- `'Back'` → `'mid-back'`, `'Lats/Lower Back/Mid Back/Upper Back'` → kebab equivalents
- `'Front/Rear/Side Delts'` → `'front/rear/side-deltoids'`, `'Rotator Cuff'` → `'rotator-cuff'`, `'Shoulders'` → `'shoulders'`
- `'Biceps/Triceps/Forearms/Quadriceps/Hamstrings/Glutes/Calves'` → lowercase
- `'Hip Flexors'` → `'hip-flexors'`, `'Inner Thighs'` → `'adductors'`, `'Tibialis'` → `'tibialis'`
- `'Traps/Neck/Core/Abs/Lower Abs/Obliques/Full Body/Legs'` → lowercase (Lower Abs collapses to abs)

The README claimed the `difficulty` field was removed; it was still present on 278 entries (and unused by any JS). Stripped from every entry.

`formatMuscleGroup` gets one new explicit entry for `'shoulders'` so the Exercises tab subtitle stays predictable.

## Change 8: Per-key error handling in loadAllData
### What changed

## Change 10: Replace inline onclick handlers with event delegation
### What changed
All 19 inline `onclick="window.gymApp.viewControllers.X.method(${id})"` interpolations across the views are gone. Each interactive element now declares its intent via `data-action` plus the relevant `data-program-id` / `data-session-id` / `data-exercise-id` / `data-exercise-index` / `data-slot` attributes. Each view installs a single delegated click listener (and where the element is `role="button"`, also a keyboard listener) that reads `data-action` and dispatches.

This drops the largest class of XSS vectors (raw IDs interpolated into JS expressions inside HTML), unlocks adding a strict CSP later, and makes the UI testable without a JS engine that evaluates inline strings.

`renderSetToggle()`'s signature changed from `(pressed, onClickExpression, ariaLabel)` to `(pressed, action, exerciseIndex, slot, ariaLabel)`.

## Change 11: Baseline security headers + CSP (Report-Only)
### What changed
`netlify.toml` now sets baseline security headers on every response:
- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: geolocation=(), microphone=(), camera=()`
- `Strict-Transport-Security: max-age=31536000; includeSubDomains`
- `Content-Security-Policy-Report-Only` — allow `'self'`, `gstatic.com`, `cdnjs.cloudflare.com` for scripts; allow `'unsafe-inline'` for now (the bundled jQuery partials loader at the bottom of `index.html` is still inline). Connect to Google APIs and Firebase RTDB / WSS endpoints. Block `object-src` entirely.

CSP is intentionally **Report-Only** at first so the deploy can't break in production while we observe whether real traffic generates violations. Once the inline script is moved to an external file (or hashed), graduate by renaming the directive to `Content-Security-Policy`.

Also added cache-control hints for the gym-tracker JS/CSS/data folders (5 min for code, 1 hour for the exercise DB) and confirmed Firebase SDK versions are already pinned (10.7.1 modular / 9.23.0 compat).

## Change 12: PWA manifest + offline service worker

## Change 14: Sync-status pill + fix sync-immediate double-pipeline
### What changed
**Two changes in one commit because they're entwined:**

1. **Sync status pill.** Tiny bottom-right indicator that shows one of:
   - **Synced** (green) — no pending writes, sync layer is healthy.
   - **Saving…** (amber) — debounced writes still in-flight, or sync layer is connecting.
   - **Offline** (gray) — `navigator.onLine` is false; we keep recording locally.

   Lives in `apps/gym-tracker/js/utils/sync-status.js`, mounted by `app.js` once during init. Polls `window.gymGetGlobalSyncStatus()` every 2 s plus listens for online/offline events. Styled in `gym-tracker.css`. Pill collapses above the bottom nav on mobile, sits in the page corner on desktop.

2. **Fixed sync-immediate's double pipeline.** `sync-immediate.js` used to keep its `localStorage.setItem` override permanently active, even after `storage-sync-robust.js` installed its own override. Every write therefore fired through both pipelines (buffer + processChange in this file, AND the queueWrite path in storage-sync). Now `sync-immediate.js` only buffers writes during the bootstrap window — once `syncSystemReady` fires, it restores the native methods so only the robust sync layer remains active. Idempotent (won't re-install if loaded twice). Replays buffered writes via the now-existent `window.syncManager.processChange` so any writes that happened before robust sync was ready still propagate.

`storage-sync-robust.js` got two small additions:
- `getGlobalStatus()` now reports `totalQueueSize` so the pill can distinguish "saving" from "synced."
- Exposes `window.gymGetGlobalSyncStatus()` so non-module code can read status.

Files touched:
- `sync-system/sync-immediate.js` (rewritten — minimal buffer with handoff)
- `sync-system/storage-sync-robust.js` (added totalQueueSize + window export)
- `apps/gym-tracker/js/utils/sync-status.js` (new)
- `apps/gym-tracker/js/app.js` (mount call)
- `apps/gym-tracker/css/gym-tracker.css` (pill styles)
- `apps/gym-tracker/sw.js` (precache the new file, bump CACHE_VERSION)
